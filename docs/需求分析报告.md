# A 股散户量化决策 —— 需求分析报告

> 本报告以"A 股个人投资者（散户）的交易决策"为研究领域，分析其真实痛点与未被满足的需求，
> 并说明本项目（A 股量化交易系统）作为解决方案的设计取舍。
>
> 报告作者是该领域的**真实用户**——本人长期参与 A 股交易，本系统最初就是为解决自己的决策问题而开发。

---

## 一、领域背景与问题定义

### 1.1 市场现状

A 股市场有 **5500+ 只股票、2 亿+ 个人投资者**。与成熟市场不同，A 股的交易结构中散户占比极高，而散户的整体收益长期跑输指数——这背后不是"运气差"，而是**决策方式的系统性缺陷**。

### 1.2 散户的核心困境

通过对自身及身边投资者的观察，散户的决策困境集中在四点：

| 困境 | 具体表现 |
|------|---------|
| **信息过载** | 5500 只股票，人工每天根本看不完；公众号、股吧、新闻信息爆炸，真假难辨 |
| **情绪化交易** | 追涨杀跌、拿不住盈利、扛单亏损——决策被情绪而非规则驱动 |
| **缺乏系统方法** | 凭"感觉"和"消息"买卖，没有可复用、可验证的方法论 |
| **没有纪律** | 明知该止损却下不了手；明知该等却忍不住——知道 ≠ 做到 |

### 1.3 真实案例（作者亲历）

报告作者曾在一只银行股上以接近当日高点的价格买入，最低佣金还吃掉了约 1% 的成本——**这正是"凭感觉择时 + 缺乏成本意识"的典型散户错误**。这类亲身踩坑，是本报告需求洞察的来源，也是开发本系统的直接动机。

### 1.4 问题本质

> 散户缺的不是"信息"，而是**"把信息变成纪律化决策的系统"**。

市场上工具很多，但要么只给信息不给决策（看盘软件），要么门槛高到散户用不动（专业量化框架）。**中间存在一个明确的产品空白。**

## 二、目标用户画像

### 2.1 核心用户

**有方法焦虑的进阶散户**——已经过了"追消息"阶段，意识到需要系统化方法，但靠自己建立不起来。

| 维度 | 画像 |
|------|------|
| 资金量 | 几万 ~ 几十万元，输得起学费但输不起本金 |
| 交易风格 | 波段 / 趋势，持仓数天到数周（非高频、非长线价投） |
| 时间 | 有正职，无法盯盘，盘后才有时间 |
| 技术能力 | 非程序员，但愿意学、愿意用工具 |
| 心理特征 | 知道自己情绪化，**想要"被纪律约束"** |

### 2.2 用户研究的真实性说明

本画像不是凭空构造——**报告作者本人就完全符合该画像**：非科班、有正职、做波段、深知自己的情绪化弱点。这种"开发者即用户"的设定，让需求判断有了第一手依据，避免了"替用户想象需求"的常见错误。

### 2.3 非目标用户

明确**不服务**以下人群，以保持产品聚焦：

- 高频 / 日内交易者（个人在延迟和成本上没有优势）
- 纯长线价值投资者（决策频率低，不需要每日扫描）
- 机构 / 专业量化团队（他们有更专业的工具与资源）

## 三、核心需求分析

将散户痛点逐条转化为产品需求，并排定优先级（P0 = 必须有，P1 = 重要，P2 = 增强）：

| 痛点 | 转化为需求 | 优先级 | 本系统的实现 |
|------|-----------|--------|-------------|
| 5500 只看不过来 | 全市场自动扫描、机械筛选 | **P0** | 数据层 + 策略层每日全量扫描 |
| 情绪化、无纪律 | 用固定策略代替"感觉"，机械产生信号 | **P0** | 4 套规则化策略，信号由代码判定 |
| 怕看漏 / 怕错过 | 主动推送，不依赖用户盯盘 | **P0** | 邮件 / 微信推送 |
| 不知道买哪只 | 对信号做质量排序与评分 | P1 | 共振池 + 五维评分 ranker |
| 大盘不好还硬做 | 市场环境过滤，弱市自动停手 | P1 | 环境过滤层（市场广度+趋势） |
| 策略到底行不行 | 用历史数据回测验证 | P1 | 回测引擎（T+1、涨跌停、手续费建模） |
| 怕被"荐股"割韭菜 | 策略透明、可理解、可自己改 | P1 | 全部策略开源、参数可配 |
| 个股黑天鹅 | 风险标的过滤（ST、退市风险等） | P2 | 板块/ST 感知，后续迭代加强 |

### 关键需求洞察

1. **"半自动"而非"全自动"**——散户既想要纪律，又不放心把钱完全交给黑箱。最优解是**系统负责"海选+排序"，人负责"最终拍板"**。
2. **透明 > 强大**——对散户而言，"看得懂、能自己改的普通策略"，比"看不懂的高级黑箱"更可信、更可用。
3. **推送是刚需不是增强**——用户最大的现实约束是"没时间盯盘"，主动推送把决策成本降到最低。

## 四、竞品分析

| 竞品 | 定位 | 优势 | 对散户的空白 |
|------|------|------|-------------|
| **同花顺 / 通达信** | 看盘工具 | 行情全、免费、用户基数大 | 只给信息不给决策；内置选股器规则初级、不成体系 |
| **vnpy** | 专业开源量化框架 | 功能强大、社区成熟 | 门槛极高——要会编程、懂架构，散户用不动 |
| **米筐 / 聚宽** | 在线量化平台 | 数据全、可在线回测 | 仍需写代码，偏机构/专业用户 |
| **荐股 APP / 公众号** | 信息服务 | 上手零门槛 | 黑箱、不可验证、利益不一致，存在割韭菜风险 |

### 竞品格局结论

市场呈现明显的**两极分化**：

```
看盘工具（同花顺）          专业框架（vnpy / 米筐）
只给信息，不给决策   ←  空白  →   能给决策，但门槛极高
        └─────── 散户卡在中间，无合适工具 ───────┘
```

**产品定位**：填补这个空白——一个**散户级、半自动、策略透明、用户可掌控**的决策辅助系统。它比看盘工具多了"决策建议"，比专业框架低了"使用门槛"。

## 五、产品方案

本系统采用六层架构，每一层精确对应前述需求：

| 层 | 对应需求 | 解决了什么 |
|----|---------|-----------|
| ① 数据层 | 信息过载 | 自动归集全市场数据，人不用自己找 |
| ② 行情层 | 信息时效 | 每日自动增量更新，数据永远是新的 |
| ③ 策略层 | 情绪化、无纪律 | 用 4 套规则化策略代替"感觉" |
| ④ 共振层 | 不知道买哪只 | 多策略叠加确认，筛出最强信号 |
| ⑤ 环境过滤层 | 大盘不好硬做 | 市场弱时自动停止推送 |
| ⑥ 输出层 | 怕看漏、没时间 | 主动推送到邮箱/微信 |

### 核心设计取舍

- **半自动定位**：系统只推信号，**不自动下单**——既给纪律，又把最终控制权留给用户，同时规避合规风险。
- **策略透明**：4 套策略全部是公开、可理解的经典技术形态（MACD、布林带、均线、趋势回踩），用户能看懂、能自己调参——刻意不做"黑箱"。
- **多维评分**：解决"5 只信号挑哪只"的真实困扰，把模糊的"哪个好"量化成 0~1 的评分。

## 六、MVP 范围与迭代路线

### 6.1 MVP（已完成并在用）

- 4 套策略 + 3 日共振池
- 市场环境过滤
- 五维评分排序
- 邮件 / 微信信号推送
- 回测引擎

> MVP 的判断标准：**能不能每天真的帮我（真实用户）做出更有纪律的决策**。答案是肯定的——系统已稳定运行，每日产出信号。

### 6.2 迭代路线

| 阶段 | 内容 | 状态 |
|------|------|------|
| 迭代 1 | 策略参数寻优（用历史回测找最优参数，替代经典默认值） | 进行中 |
| 迭代 2 | 基本面风险过滤（剔除业绩暴雷、退市风险标的） | 规划中 |
| 迭代 3 | 可视化看板（信号、回测曲线、持仓的图形化呈现） | 规划中 |

### 6.3 刻意"不做"的事（边界）

产品克制同样重要：

- **不做自动下单**——合规风险 + 把控制权留给用户
- **不做高频交易**——个人在延迟和成本上没有优势，不在不擅长的领域竞争
- **不做收益承诺**——策略盈利性需长期实盘检验，系统的价值是"提供纪律化决策框架"，而非"保证赚钱"

---

## 七、总结

本报告完整走了一遍 **"发现需求 → 定义用户 → 拆解痛点 → 分析竞品 → 设计方案 → 规划迭代"** 的产品流程。

核心结论：

> A 股散户真正缺的，是一个**介于"看盘工具"与"专业框架"之间、半自动、透明、可掌控**的决策辅助系统。本项目正是面向这一空白的解决方案——它从作者自身的真实痛点出发，目标是把"知道但做不到"的散户决策，变成"系统辅助下可执行的纪律"。

而这套系统本身，是由一名**非科班开发者借助 AI 编程工具从零实现**的——这意味着：**发现需求的人，也具备把需求落地成产品的能力**。对一个产品工程师岗位而言，这正是"需求洞察 + 工程实现"的完整闭环。
