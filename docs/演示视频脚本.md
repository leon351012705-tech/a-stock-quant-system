# 演示视频脚本（约 2 分钟）

> 用途：录一段操作演示视频，放到 GitHub Release 或 README，替代"可下载安装包"。
>
> 录制工具建议：Windows 自带「Xbox Game Bar」（Win+G）、OBS Studio，或任何录屏软件。
> 旁白可以录音，也可以后期配字幕。**照着下面的「旁白」念即可。**

---

## 录制前准备

1. 关掉无关窗口，桌面整洁
2. 终端字体调大一点（演示用，看得清）
3. 提前打开：① 终端（定位到项目目录）② 邮箱（能看到信号邮件）③ 代码编辑器或 GitHub 仓库页
4. 确认当天数据已更新（这样演示流畅，不卡在拉数据）

---

## 分镜脚本

### 镜头 1 ｜ 开场（约 10 秒）

**画面**：GitHub 仓库首页 / 或 README 顶部

**旁白**：
> "这是我用 AI 编程工具，从零独立开发的一套 A 股量化交易系统。下面演示它每天是怎么跑的。"

---

### 镜头 2 ｜ 数据更新（约 20 秒）

**画面**：终端执行
```bash
python run_daily_update_v2.py
```

**旁白**：
> "第一步，自动更新行情数据。它会从数据源拉取全市场五千多只股票的当日数据。如果之前漏了几天，它会自动补齐；遇到节假日会自动跳过——这些容错都是做过工程处理的。"

**要点**：让终端日志滚动出来，展示"自动补 gap""节假日识别"之类的输出。

---

### 镜头 3 ｜ 信号扫描（约 45 秒）

**画面**：终端执行
```bash
python run_signal_scan.py
```

**旁白**：
> "第二步，全市场信号扫描。系统会先判断当天的大盘环境——如果市场偏弱，就不推送信号。"
>
> "然后用四套策略并行扫描全部股票：MACD 金叉、布林带超跌反弹、均线多头排列、趋势回踩。"
>
> "命中多个策略的股票会进入'共振池'，再经过一个五维评分模型排序——信号新鲜度、共振强度、趋势、流动性、风险，最后给每只票打一个分。"

**要点**：让扫描进度、市场环境判断、共振池结果都在终端里滚出来。

---

### 镜头 4 ｜ 信号邮件（约 25 秒）

**画面**：切换到邮箱 / 手机，打开收到的信号邮件

**旁白**：
> "扫描结果会自动推送到邮箱。这是今天的信号邮件——按板块分类，每只股票带星级评分和风险标签。优先看的是'共振池'里评分高的票。"
>
> "整个流程从拉数据到出信号，大约 80 秒，每天盘后一键运行。"

**要点**：邮件里如有真实持仓个股，可以打码；展示排版和评分即可。

---

### 镜头 5 ｜ 代码与架构（约 15 秒）

**画面**：快速翻一下项目结构 / README 的架构图

**旁白**：
> "整个系统是六层架构，约一万行代码。从数据层、策略层到信号推送，每一层职责单一、可独立替换。代码已全部开源在 GitHub。"

---

### 镜头 6 ｜ 收尾（约 10 秒）

**画面**：回到 README 或 GitHub 仓库页

**旁白**：
> "这套系统是我作为非科班开发者，借助 AI 工具独立完成的。它解决的是我自己作为散户的真实决策问题。详细的需求分析和开发实录都在仓库文档里，谢谢观看。"

---

## 录制小贴士

- **节奏**：终端日志滚动快的话，可以适当加速剪辑，别让观众等
- **时长**：控制在 2 分钟以内，招聘官时间有限
- **画质**：1080p 即可，字幕比纯语音更稳妥（环境吵也能看）
- **发布**：传 B 站 / GitHub Release，README 里放链接

---
